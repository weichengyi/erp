# 宝塔面板 Docker 一键部署指南

> 不懂技术也能 5 分钟部署！完全图形界面操作！

---

## 🎯 适用场景

- ✅ 你有一台服务器（阿里云/腾讯云/华为云等）
- ✅ 服务器安装了宝塔面板
- ✅ 你不懂技术，不会用命令行
- ✅ 你想一键部署这个 ERP 系统

---

## 📋 前置要求

### 1. 安装宝塔面板

如果你还没有宝塔面板：

**访问**: https://www.bt.cn/download/

选择你的服务器系统，复制安装命令，在服务器执行即可。

### 2. 安装 Docker

在宝塔面板中：
1. 登录宝塔面板（通常是 `http://你的服务器 IP:8888`）
2. 点击左侧 **"软件商店"**
3. 搜索 **"Docker"**
4. 点击 **"安装"**

等待 3-5 分钟，Docker 安装完成。

### 3. 安装 Docker 管理器（重要！）

在宝塔面板中：
1. 点击左侧 **"软件商店"**
2. 搜索 **"Docker 管理器"** 或 **"容器管理"**
3. 点击 **"安装"**

这个插件让你用图形界面管理 Docker，不需要敲命令！

## 🚀 宝塔面板部署步骤

### 步骤 1：下载项目代码

**方法 A：使用 Git 拉取（推荐）**

1. 登录宝塔面板
2. 点击左侧 **"文件"**
3. 进入 `/www/wwwroot/` 目录
4. 点击 **"创建文件夹"**，命名为 `erp-trade`
5. 进入 `erp-trade` 文件夹
6. 点击 **"终端"** 或 **"Git"** 按钮
7. 执行命令：
   ```bash
   git clone https://github.com/weichengyi/erp.git .
   ```
8. 等待代码下载完成

**方法 B：手动上传（如果不会 Git）**

1. 在电脑上下载项目 ZIP 包
2. 登录宝塔面板
3. 点击左侧 **"文件"**
4. 进入 `/www/wwwroot/`
5. 创建文件夹 `erp-trade`
6. 点击 **"上传"**
7. 上传 ZIP 文件并解压

---

### 步骤 2：配置 Docker Compose

1. 在宝塔面板文件管理中，进入 `/www/wwwroot/erp-trade`
2. 找到文件 `docker-compose.yml`
3. 点击 **"编辑"**
4. **不需要修改任何内容**，直接保存即可

> ⚠️ 默认配置已经设置好，不需要修改！

---

### 步骤 3：使用 Docker 管理器部署

#### 方式一：使用 Docker Compose（推荐）⭐

1. 在宝塔面板左侧菜单，找到 **"Docker"** 或 **"容器管理"**
2. 点击 **"Compose"** 或 **"编排"** 标签
3. 点击 **"添加"** 或 **"新建"**
4. 填写信息：
   - **名称**: `erp-trade`
   - **描述**: ERP 系统
   - **路径**: `/www/wwwroot/erp-trade`
   - **文件**: `docker-compose.yml`
5. 点击 **"确定"** 或 **"创建"**
6. 点击 **"启动"** 或 **"运行"** 按钮
7. 等待 3-5 分钟，容器启动完成

#### 方式二：手动创建容器（如果 Compose 不可用）

如果宝塔面板没有 Compose 功能，手动创建 3 个容器：

**1. 创建 MySQL 容器**

在 Docker 管理器中：
1. 点击 **"镜像"** → **"拉取镜像"**
2. 输入 `mysql:8.0`，点击拉取
3. 等待拉取完成
4. 点击 **"容器"** → **"添加容器"**
5. 填写：
   - **镜像**: `mysql:8.0`
   - **容器名称**: `erp-mysql`
   - **端口映射**: `3306:3306`
   - **环境变量**:
     ```
     MYSQL_ROOT_PASSWORD=erp_root_2026
     MYSQL_DATABASE=erp_trade
     MYSQL_USER=erp_user
     MYSQL_PASSWORD=erp_password_2026
     ```
   - **存储卷**: 添加一个持久化存储
6. 点击 **"确定"** 创建

**2. 创建后端容器**

1. 点击 **"镜像"** → **"构建镜像"**
2. 填写：
   - **镜像名称**: `erp-backend`
   - **Dockerfile 路径**: `/www/wwwroot/erp-trade/Dockerfile`
3. 点击 **"构建"**，等待 5-10 分钟
4. 构建完成后，点击 **"容器"** → **"添加容器"**
5. 填写：
   - **镜像**: `erp-backend`
   - **容器名称**: `erp-backend`
   - **端口映射**: `8080:8080`
   - **环境变量**:
     ```
     SPRING_DATASOURCE_URL=jdbc:mysql://erp-mysql:3306/erp_trade?...
     SPRING_DATASOURCE_USERNAME=erp_user
     SPRING_DATASOURCE_PASSWORD=erp_password_2026
     ```
   - **网络**: 选择与 MySQL 相同的网络
6. 点击 **"确定"** 创建

**3. 创建前端容器**

1. 点击 **"镜像"** → **"构建镜像"**
2. 填写：
   - **镜像名称**: `erp-frontend`
   - **Dockerfile 路径**: `/www/wwwroot/erp-trade/frontend/Dockerfile`
3. 点击 **"构建"**，等待 5 分钟
4. 构建完成后，点击 **"容器"** → **"添加容器"**
5. 填写：
   - **镜像**: `erp-frontend`
   - **容器名称**: `erp-frontend`
   - **端口映射**: `80:80`
6. 点击 **"确定"** 创建

---

### 步骤 4：验证部署

1. 在 Docker 管理器中，查看容器列表
2. 确保 3 个容器都是 **"运行中"** 状态

## 🛠️ 常用操作（图形界面）

### 查看日志

1. 在 Docker 管理器中，点击 **"容器"**
2. 找到对应的容器（如 `erp-backend`）
3. 点击 **"日志"** 或 **"Log"** 按钮
4. 查看运行日志

### 重启容器

1. 在容器列表中，找到要重启的容器
2. 点击 **"重启"** 按钮
3. 等待重启完成

### 停止容器

1. 在容器列表中，找到要停止的容器
2. 点击 **"停止"** 按钮

### 启动容器

1. 在容器列表中，找到已停止的容器
2. 点击 **"启动"** 按钮

### 删除容器

1. 在容器列表中，找到要删除的容器
2. 点击 **"删除"** 按钮
3. 确认删除

## ❓ 常见问题

### 问题 1: 容器启动失败

**解决**:
1. 在 Docker 管理器中查看容器日志
2. 检查端口是否被占用（80, 8080, 3306）
3. 检查服务器内存是否足够（至少 2GB）

### 问题 2: 无法访问系统

**解决**:
1. 检查服务器防火墙是否开放端口
2. 在宝塔面板 **"安全"** 中放行端口：80, 8080
3. 检查容器是否正常运行

### 问题 3: 数据库连接失败

**解决**:
1. 确保 MySQL 容器先启动
2. 等待 1 分钟让 MySQL 初始化完成
3. 重启后端容器

### 问题 4: 内存不足

**解决**:
1. 在宝塔面板查看内存使用情况
2. 关闭不必要的容器或服务
3. 升级服务器配置（建议 2GB 以上内存）
